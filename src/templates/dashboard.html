How it works

When you package content in the sandbox:

  1. Key Agreement: The system runs a simulated ML-KEM-768 encapsulation exchange, deriving a shared symmetric session key.
  2. Data Protection: Data payloads are encrypted using the derived shared secret, protecting against "Store Now, Decrypt Later" scenarios.
  3. Authenticity: An ML-DSA-65 signing block is generated, binding the data hash to a lattice public key to prevent tampering.

Note: The downloaded .pqc file contains the JSON encapsulation package, complete with simulated cryptographic hex headers (shared secrets, signatures, and public keys).
